/*
 * 命令模式【Command】:
 * 将请求或动作封装在对象中，以便在适当的时候调用。
 * 
 * 1. 命令接口： 提供命令操作的接口【Command】
 * 2. 命令对象： 实现命令操作的对象【SoftCommand】
 * 3. 接受者：      知道如何执行命令对象【HardCommand】
 * 4. 调用者：     调用命令操作的上下文 【Client】
 *
 * 说明：
 * 对方法运行的常规做法是调用它，这是一种主动调用法，即程序员能够确定调用的时机。
 * 然而，在有些场合，程序员并不知道方法何时被调用，因此，将请求封装在对象中，
 * 当客户端需要调用方法时，由系统来自动调用。
 *
 * 命令模式的典型应用是菜单设计。
 *
 * 程序员并不知道哪个菜单被执行，因此，也不知道哪个方法会被执行；
 * 因此，只能将菜单与命令对象绑定起来，当点击菜单时，就执行相应的命令对象。
 *
 *
 */
